Chapter 1 --> What is HTML?

HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It defines the structure of web content.

Think of it like: The skeleton of a building. It provides the structure, while CSS provides the paint and JS provides the elevators and lighting.

The Browser's Job

When you open a website, your browser reads the HTML from top to bottom. It creates something called the DOM (Document Object Model), which is a tree-like map of your page that JavaScript can interact with later.

Chapter 2 --> Elements & Tags

The Boilerplate Structure

Every HTML document must follow a specific boilerplate structure to be valid.

index.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Website</title>
</head>
<body>
    <!-- Content goes here -->
    <h1>Hello World!</h1>
</body>
</html>

Live Result:

Hello World!

Key Parts:

Element Description
<!DOCTYPE> Tells the browser this is an HTML5 document.
<html> The root element of the page.
<head> Contains metadata (title, link to CSS, meta tags).
<title> Defines the title shown in the browser's tab.
<body> Contains the visible content of the page.

Chapter 5 --> Links & Images

Connectivity is what makes the web "HyperText". Use <a> for links and <img> for visual content. 

<!-- External Link -->
<a href="https://google.com" target="_blank">Visit Google</a>

<!-- Image -->
<img src="path/to/image.jpg" alt="Description of image" width="300">

Live Result:

Visit Google

(Showing your hero image as a sample)

SEO TIP: Always use the alt attribute on images. If the image is just for decoration, use an empty alt="" so screen readers ignore it.

Relative vs. Absolute Paths

Absolute: Full URL (e.g., https://google.com/logo.png). Use for external sites.

Relative: Link to a file inside your own folder (e.g., images/photo.jpg). Best practice for your own assets!

Chapter 6 --> Forms & Interactive Elements

Forms are used to collect user input. Modern HTML provides many specialized input types.

<form action="/submit" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>

    <label for="email">Email:</label>
    <input type="email" id="email">

    <button type="submit">Send</button>
</form>

Live Result:

GET vs. POST Methods

When you send a form, the method attribute decides how the data travels:

  • GET: Puts data in the URL (e.g., site.com?name=John). Visible to everyone!
  • POST: Hides data inside the request body. Safest for passwords and private info.

The "Placeholder" Attribute: Use placeholder="Type name here..." to show a faint hint inside the box.

Chapter 10 --> Interview Prep: Ultimate HTML Question Bank

Prepare for your next frontend interview with these 30+ comprehensive HTML questions and detailed answers.

LEVEL: BEGINNER

Q1: What is HTML?

Detailed Answer: HTML stands for HyperText Markup Language. It is the standard markup language used to create the structure of web pages. It uses "markup" to annotate text, images, and other content for display in a web browser.

Q2: What is the difference between HTML elements and tags?

Detailed Answer: An element is the entire unit: the opening tag, the content, and the closing tag. A tag is just the syntax used to mark the beginning and end of an element (e.g., <p> is a tag, <p>Hello</p> is an element).

Q3: What is the purpose of the <!DOCTYPE html> declaration?

Detailed Answer: It is an instruction to the web browser about what version of HTML the page is written in. <!DOCTYPE html> specifically triggers "Standards Mode" for HTML5 documents.

Q4: What are 'void elements' in HTML?

Detailed Answer: Void elements (also called self-closing tags) are elements that do not contain any content and therefore do not need a closing tag. Examples include <img>, <br>, <hr>, and <input>.

Q5: What is the purpose of the 'alt' attribute in images?

Detailed Answer: The alt attribute provides alternative text if an image cannot be displayed. It is also essential for accessibility, as screen readers read this text to visually impaired users.

Q6: What is the difference between <div> and <span>?

Detailed Answer: <div> is a block-level element (starts on a new line, takes full width), whereas <span> is an inline element (stays in line, takes only necessary width).

Q7: What is Semantic HTML?

Detailed Answer: Semantic HTML uses tags that clearly describe their meaning to both the browser and the developer (e.g., <header>, <article>, <footer>), rather than generic tags like <div>.

Q8: What is the purpose of the <head> tag?

Detailed Answer: It contains metadata about the page, such as the title, character set, links to CSS/JS files, and SEO descriptions. It is not displayed on the page itself.

Q9: What are HTML attributes?

Detailed Answer: Attributes provide additional information about an element. They are always specified in the opening tag and usually look like name="value" (e.g., href="google.com").

Q10: What is the difference between <b> and <strong>?

Detailed Answer: <b> is for stylistic bolding, while <strong> indicates that the text has strong importance, which helps with SEO and accessibility.

LEVEL: INTERMEDIATE

Q11: What is the purpose of the <label> tag?

Detailed Answer: It provides a label for a form element. Clicking the label also focuses/activates the associated input, improving the user experience and accessibility.

Q12: What is the difference between GET and POST form methods?

Detailed Answer: GET appends data to the URL (visible and limited size), while POST sends data in the request body (private and unlimited size). Use POST for sensitive info like passwords.

Q13: What is the 'target="_blank"' attribute used for?

Detailed Answer: It is used in anchor tags (<a>) to open the linked document in a new browser tab or window.

Q14: What are HTML Entities?

Detailed Answer: Entities are used to display reserved characters (like < or >) or special symbols (like ©). Example: &lt; displays as <.

Q15: How do you create an Unordered List vs an Ordered List?

Detailed Answer: Use <ul> for bullet points and <ol> for numbered/lettered lists. Individual items always go in <li> tags.

Q16: What is an Iframe?

Detailed Answer: An <iframe> is an "Inline Frame" used to embed another HTML document within the current page (e.g., a YouTube video or Google Map).

Q17: What is the purpose of 'rel="noopener noreferrer"'?

Detailed Answer: It is a security best practice when using target="_blank" to prevent the new page from accessing the original page via the window.opener object.

Q18: What is a nested list?

Detailed Answer: It is a list within another list. It's useful for creating hierarchical structures like navigation menus or table of contents.

Q19: What is the difference between <header> and <h1>?

Detailed Answer: <header> is a semantic container for introductory content (logo, nav), while <h1> is the highest level heading on a page.

Q20: How do you add an image in HTML?

Detailed Answer: Using the <img> tag with src (path) and alt (description) attributes. Example: <img src="pic.jpg" alt="Description">.

LEVEL: ADVANCED

Q21: What is the difference between SVG and Canvas?

Detailed Answer: SVG is vector-based (XML) and stays sharp at any size. Canvas is pixel-based (bitmap) and better for high-performance graphics and games using JavaScript.

Q22: What is the difference between async and defer in script tags?

Detailed Answer: Both load scripts asynchronously. async executes the script immediately after download (blocking HTML), while defer waits until the HTML is fully parsed.

Q23: What are 'data-' attributes?

Detailed Answer: They allow you to store custom data private to the page or application inside standard HTML elements, which can then be easily accessed via JavaScript.

Q24: What is the difference between LocalStorage and SessionStorage?

Detailed Answer: Both store data in the browser. LocalStorage lasts forever (until cleared), while SessionStorage is cleared when the browser tab is closed.

Q25: What is the purpose of the <main> tag?

Detailed Answer: It identifies the primary unique content of a document. It should not contain sidebars, navigation links, or logos that are repeated across pages.

Q26: What is a Web Worker?

Detailed Answer: A script that runs in the background of a web page, allowing for heavy computations without freezing the main UI thread.

Q27: What is the purpose of the <figure> and <figcaption> tags?

Detailed Answer: They are used to group media (like images) with a descriptive caption. This provides semantic structure and better context for accessibility.

Q28: What is Responsive Web Design in the context of HTML?

Detailed Answer: In HTML, this mainly involves using the <meta name="viewport" content="..."> tag and semantic structure that CSS can easily adapt to different screens.

Q29: What are WebSockets?

Detailed Answer: An HTML5 technology that allows for a full-duplex communication channel between a browser and a server (perfect for real-time apps like chats).

Q30: What is the Geolocation API in HTML5?

Detailed Answer: An API that allows the user to provide their location to the web application, enabling features like finding nearby stores or local weather.

Q31: What is ARIA in web development?

Detailed Answer: ARIA (Accessible Rich Internet Applications) is a set of attributes that you can add to HTML elements to make them more accessible to people using assistive technologies like screen readers.
